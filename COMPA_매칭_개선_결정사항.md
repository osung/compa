# COMPA 매칭 — 최종 선택 정리

기업 진성수요(78건) ↔ 국가 R&D 과제(약 96만 건) 매칭 파이프라인과, 검색 리콜
개선 실험을 통해 도출한 **최종 결정 사항**을 정리한다.

---

## 1. 매칭 파이프라인 (현행 확정)

- 실행: `compa_match.py` (8개 모듈 통합 단일 스크립트) / Colab: `compa_colab.ipynb`
- 흐름: **키워드 추출(LLM) → pro-sroberta 임베딩 → 단일 SBERT top100 → LLM 적합도(0~100) → top5 → 4섹션 상세근거**
- LLM: **Qwen 3.x 35B-A3B** (Apple Silicon=MLX / CUDA=vLLM)
- 전체 실행 실측: **78건 = 146분(~2.4h)**, 수요당 ~108~119초 (top100 기준)
- 결과: 담당자별 `COMPA_{담당자}_최종추천.pkl/xlsx` (78수요 × Top5 = 390행)

## 2. 수동(구글시트) 매칭 통합 — 확정

구글시트의 전문가 수동 매칭(27개 수요)과 자동 매칭을 비교·통합.

- 27개 수요 비교 → **시트가 더 나은 11건** 선별 → 시트 과제 31건 추가(`수동=Y`)
- 추가분을 **우리 LLM으로 재채점** → top1초과 4건 / top5진입 19건 / 미달 8건
- **미달 8건 제거** + **각 수요를 점수순 top5로 정리**(수동이 낮은 자동을 대체)
- **최종 데이터셋: `COMPA_전체_최종추천_수동보완.pkl/xlsx`** (390행, `수동`·`수동_평가` 컬럼, 최종 수동 매칭 17건)

**우리 top1을 능가한 수동 매칭 4건** (LLM 검증):
| 수요 | 기업 | 수동 점수 | 우리 top1 |
|:---:|---|:---:|:---:|
| 70 | 기업 A | 95 | 45 |
| 61 | 기업 B | 85 | 75 |
| 64 | 기업 C | 85 | 75 |
| 66 | 기업 D | 85 | 75 |

## 3. 수동 매칭이 자동에서 누락된 원인 (진단 확정)

수동 매칭 17건 중:
- **16건 = SBERT top100 밖** (임베딩 순위 중앙값 **90,905위**) — 키워드↔과제설명문 **문체 불일치**가 근본 원인
- 1건 = top100 내였으나 LLM 점수 top5 미달

→ **병목은 1차 필터(임베딩 top100)의 리콜**. LLM이 낮게 본 게 아님.

## 4. 검색 리콜 개선 실험 — 방법별 비교 (놓친 16건 회수 기준)

| 방법 | top100 | top200 | 시간(78건) | 비고 |
|---|:---:|:---:|---|---|
| 임베딩 단독(현행) | 0/16 | 0/16 | — | 정답이 9만 위로 밀림 |
| **top-N 확대** | — | — | — | ❌ 전건 포함에 top~60만(코퍼스 62%) 필요 → 폐기 |
| BM25(일반) | ~2/16 | ~2-3/16 | 인덱스 31s+검색 12ms/건 | 순위 중앙값 9만→**722위**로 급개선 |
| BM25 RRF 하이브리드 | 1/16 | — | 위와 동일 | ❌ 임베딩 순위가 나빠 RRF 희석 |
| BM25 후보풀 합집합 | — | — | LLM 6~11배↑ | top1000→10/16, top3000→12/16 (LLM 비용이 병목) |
| BM25F(필드가중 3:2:1) | 4/16 | 5/16 | 인덱스 34s | 제목형↑ but 설명문형↓ (전역 가중 한계) |
| **BM25 top3000 → 크로스인코더** | **6/16** | **8/16** | 재랭킹 ~1.7h | ⭐ 최고. 제목·설명문형 모두 회수 |

### 크로스인코더 pool별 (재랭커 = `BAAI/bge-reranker-v2-m3`, Apple GPU)
| pool | 1건 재랭킹 | 78건 재랭킹 | 회수(16건) |
|---|---|---|---|
| top3,000 | 80초 | ~1.7h | 8/16 |
| top5,000 | 157초 | ~3.4h | +수요63 포함 가능 |
| top10,000 | 415초(6.9분) | ~9.0h | **+1건(수요63)뿐 → ROI 낮음** |

(1회성 준비 ~40초: 코퍼스 로드 3s + BM25 인덱스 32s + 재랭커 로드 6s)

### HyDE(가상 키워드 생성 → 임베딩) — 검증 후 채택 안 함
우리 임베딩은 '키워드 공간'이라 HyDE도 가상 '키워드'를 생성해 적용(최종 수동 17건, pool별 회수):
| pool | 임베딩(현행) | BM25 | HyDE | HyDE∪BM25 |
|---|:---:|:---:|:---:|:---:|
| 200 | 1/17 | 2/17 | 2/17 | 4/17 |
| 500 | 1/17 | 7/17 | 2/17 | 8/17 |
| 1000 | 3/17 | 10/17 | 2/17 | 11/17 |
| 2000 | 3/17 | 11/17 | 3/17 | 12/17 |
| 3000 | 3/17 | 12/17 | 3/17 | 13/17 |
- HyDE 단독 2~3/17로 현행 임베딩과 차이 없음. HyDE∪BM25도 BM25 단독 대비 **+1건뿐**.
- 원인: HyDE 가상 키워드가 상위개념으로 쏠려 **잘 잡히던 것마저 순위 급락**(예: 수요68 533→285,981위, 수요75 26→51위). **키워드 공간에선 BM25(정확 용어 일치)가 이미 우위** → HyDE 채택 안 함.

### 35B LLM 채점 비용(참고, 실측)
- 배치(20후보) 5.8초, 상세근거 1건 7.8초
- 후보 top100=1건 ~68초, top200 ~100초, **top10000=1건 ~49분 / 78건 ~64h**
- → LLM은 반드시 **top200 이하로 고정**해야 함

## 5. 최종 선택 (권장 파이프라인)

> **BM25 top3000 → 크로스인코더 재랭킹(bge-reranker-v2-m3) → top200 → 35B LLM 정밀 채점 → top5**

- **후보 생성**: BM25(사실상 무료) — 리콜 담당
- **재랭킹**: 크로스인코더로 실제 관련도 재정렬 → top200 (제목형/설명문형 모두 회수)
- **정밀 채점**: 35B LLM은 **top200 고정**(시간 예산 유지)
- **pool 크기**: **top3000 확정**. (top5000은 수요63 1건 추가 회수하나 재랭킹 시간 2배, top10000은 +6시간에 +1건뿐이라 모두 미채택)
- 시간 개요(78건): BM25 무시 + 재랭킹 ~1.7h + 35B(top200) ~2.2h ≈ **약 4시간**

### 회수 성능 (16건 기준)
- 현행(임베딩 top100): 0/16
- **선택안(BM25 top3000 + 재랭커 + LLM top200): 8/16 top200 진입**, 아깝게 밀린 4건은 LLM 최종 판정에서 추가 회수 여지

---

## 6. 남은 과제 / 검증 결과

- **HyDE 검증 완료 → 채택 안 함**: 키워드 공간에서 HyDE는 실효 낮음(4절 표). 리콜은 BM25가 대부분 담당.
- **키워드 임베딩의 근본 한계로 남는 소수**: 어휘 겹침이 약한 수요 61(암생존자 식이), 66(진동감쇄), 71(세라믹코팅)은 임베딩·BM25·HyDE 모두로 top3000 회수 실패.
  - 향후 개선 방향(추가 연구): 임베딩 모델/문서표현 개선(과제 설명문 기반 재임베딩), 멀티쿼리 확장 등.
